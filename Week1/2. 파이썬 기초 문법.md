# 1. 변수의 정의와 동적 타이핑 (Variables and Dynamic Typing)

## 1.1 변수의 개념적 이해
- 프로그래밍에서 변수 (Variable) 란 단순히 값을 저장하는 바구니가 아니라, 메모리 상에 존재하는 특정 객체 (Object) 에 붙여진 이름표와 같음
- 파이썬은 변수가 값 자체가 아닌 객체의 주소를 가리키는 참조 (Reference) 방식을 사용함
- 오.. C++에서 포인터처럼 값의 주소를 참조하는 그런 느낌일까?

## 1.2 동적 타이핑 (Dynamic Typing)의 메커니즘
- 파이썬은 변수를 선언할 때 자료형을 명시하지 않음
- 값이 할당되는 시점(Runtime)에 해당 데이터의 타입을 결정하는데, 이를 동적 타이핑이라 한다
- 이는 코드의 유연성을 제공하지만, 의도치 않은 타입 변화로 인한 논리적 오류를 발생시킬 수 있으므로 주의가 필요함
- 흠.. C, C++ 에서는 어떤 변수를 만들 때 Int, Float 등 반드시 어떤 타입인지 표기를 해줘야하긴 했었지. 

```
# 1. 변수 할당과 참조의 변화
data = 100 # 정수형 객체 생성 및 data 변수가 이를 참조
print(f"초기 타입: {type(data)}")

data = "Algorithm" # 이제 data는 문자열 객체를 참조함 (타입이 유연하게 변경됨)
print(f"변경 후 타입: {type(data)}")

# 2. 참조에 의한 객체 공유 (중요 개념)
list_a = [10, 20, 30]
list_b = list_a # 동일한 메모리 주소를 공유함

list_b.append(40) # b를 수정했으나
print(f"list_a의 상태: {list_a}")  # a도 함께 변경됨: [10, 20, 30, 40]
```

## 1.3 용어 해설
- 객체 (Object) : 파이썬에서 모든 데이트 (숫자, 문자, 리스트 등)는 객체로 취급. 각 객체는 고유한 ID, 타입, 값을 가짐
- 참조 (Reference) : 변수가 객체의 메모리 주소를 가리키고 있는 상태를 의미

# 2. 데이터 형 변환과 예외 처리 (Type Casting)
## 2.1 명시적 형 변환의 필수성
- 코딩 테스트 환경에서는 서로 다른 타입의 데이터가 혼용되는 경우가 많음
- 특히 표준 입력으로 들어오는 데이터는 기본적으로 문자열이므로, 이를 수치 연산에 사용하려면 반드시 명시적 형 변환을 수행해야 한다
```
# 사용자의 입력을 정수로 변환하는 예시
new_input = "25 # 문자열 데이터
age = int(raw_input) # 정수형으로 명시적 변환
next_year_age = age + 1 # 이제 산술 연산이 가능함

# 수치형 간의 변환
pi = 3.14159
integer_pi = int(pi) # 소수점 아래를 버리고 정수 3만 취함
print(f"정수 변환 결과: {integer_pi}")
```

## 2.2 형 변환 시 발생할 수 있는 오류 (ValueError)
- 숫자로 변환할 수 없는 문자열(ex: "abc) 을 int() 함수에 전달하면 프로그램이 중단됨
- 이를 방지하기 위해 예외 처리를 만들어주는게 필요하다

```
user_data = "2024년"
try:
    # 숫자가 아닌 문자가 포함되어 있어 에러 발생 가능성이 높음
    converted_data = int(user_data)
except ValueError:
    # 에러 발생 시 프로그램 종료 대신 대체 로직 수행
    print("수치형 데이터로 변환할 수 없는 형식입니다. 전처리가 필요합니다.")
```

# 3. 연산자 체계와 연산 효율성 (Operators)
## 3.1 산술 연산자와 정밀한 활용
- 파이썬은 나눗셈 연산을 용도에 따라 세분화하여 제공
- 특히 인덱스 계산이나 순환 구조를 설계할 때 핵심적으로 사용됨

```
# 나눗셈 연산의 구분
total = 10
count = 3

print(f"실수 나눗셈: {total / count}")   # 3.333... (정밀한 결과)
print(f"몫 연산(//): {total // count}")  # 3 (소수점 버림, 인덱스 계산 시 주로 사용)
print(f"나머지 연산(%): {total % count}") # 1 (짝홀 판별, 주기성 계산에 필수)

# 연산자 우선순위 활용
# 곱셈과 나눗셈이 덧셈보다 먼저 계산되나, 가독성을 위해 괄호를 권장함
result = (total + 5) * (count ** 2) # 15 * 9 = 135
```

## 3.2 논리 연산과 단락 평가 
- 논리 연산자 and, or 는 조건문의 효율성을 극대화한다
- 앞의 조건만으로 결과가 확정되면 뒤의 조건은 실행하지 않는 특성이 있다

```
# 단락 평가의 실전 사례
def check_status():
    print("시스템 상태 확인 중...")
    return True

# list_a 가 비어있지 않을 때만 두번째 조건을 확인하므로 에러를 방지함
items = []
if items and check_status():
    print("아이템이 존재하며 시스템이 정상입니다")
else:
    print("아이템이 없거나 확인을 생략하였습니다") # check_status() 는 실행조차 안됨
```


# 4. 제어문을 통한 논리 흐름 설계
## 4.1 조건문의 논리적 배타성
- if-elif-else 구문은 상호 배타적인 조건을 처리할 때 사용함
- 조건의 순서는 가장 구체적인 것부터 일반적인 순서로 배치하는 것이 논리적으로 타당하다

```
score = 88

if score >= 90:
    print("최우수 등급입니다")
elif score >= 70:
    # 이미 90점 이상은 위에서 걸러졌으므로 70~89점 사이임을 보장함
    print("우수 등급입니다.")
else:
    print("노력이 더 필요합니다.")
```
## 4.2 반복문과 range 객체의 성능
- for문은 시퀀스 자료형의 요소를 순차적으로 순회한다
- 이 때 사용하는 range는 모든 숫자를 한꺼번에 메모리에 올리지 않고, 순회할 때마다 숫자를 하나씩 생성하는 효율적인 구조를 가진다

```
# 1부터 100까지의 합 구하기
total_sum = 0
for i in range(1,101):
    total_sum += i

# 2차원 리스트(행렬) 순회 예시
matrix = [[1,2], [3,4], [5,6]]
for row in matrix:
    for element in row:
        print(element, end=" ") # 1 2 3 4 5 6 순차 출력
```

# 문자열 및 리스트 자료구조와 조작 
## 5.1 가독성 중심의 문자열 포맷팅
- 현태 파이썬에서 가장 권장되는 방식은 f-string
- 변수 뿐만 아니라 간단한 연산식도 포함할 수 있어 코드가 매우 간결해진다

```
name = "Student"
rank = 1
# 가독성이 매우 뛰어난 f-string 방식
message = f"안녕하세요, {name}님! 현재 당신의 순위는 {rank}위 입니다."
```

## 5.2 리스트의 동적 관리와 시간 복잡도
- 리스트는 순서가 있는 가변 객체
- 코딩 테스트에서는 특정 메서드의 실행 속도를 인지하고 사용하는 것이 성능 최적화의 핵심이다

```
# 리스트 기본 조작 및 효율성
numbers = []
numbers.append(10) # 리스트 끝에 추가: O(1) - 매우 빠름
numbers.append(20)
numbers.append(30)

# 특정 위치의 데이터 삭제 및 반환
last_item = numbers.pop() # 30 추출: O(1)

# 리스트의 길이 확인
current_size = len(numbers) # 2: O(1) - 파이썬은 길이를 따로 저장해 둠

# 인덱싱과 슬라이싱
print(f"첫 요소: {numbers[0]}, 마지막 요소: {numbers[-1]}")
```
