# 1. 리스트 컴프리헨션과 데이터 필터링
## 1.1 성능적 이점과 내부 동작
- 리스트 컴프리헨션은 파이썬다운 코드의 정수이다
- 단순히 코드를 한 줄로 줄여주는 가독성만 가지는 것이 아니다
- 일반적인 for 루프에서 append 메서드를 호출하면 매번 함수를 불러오고 리스트의 크기를 조정하는 오버헤드가 발생하지만, 리스트 컴프리헨션은 파이썬 인터프리터 내부의 C 언어 수준에서 최적화되어 실행되므로 훨씬 빠른 연산 속도를 보장한다

```
# [미션 1] 짝수 추출 및 제곱 처리
# 목표: 입력 리스트에서 짝수만 골라 제곱한 결과를 반환한다.
numbers = [1, 2, 3, 4, 5, 6]

# 리스트 컴프리헨션 구조 분해:
# [ (표현식) for (항목) in (반복가능객체) if (조건문) ]
# 1. 항목(n): numbers의 요소를 하나씩 꺼내온다.
# 2. 조건문(if n % 2 == 0): 짝수인 경우에만 다음 단계로 진행한다. (필터링)
# 3. 표현식(n**2): 필터링된 숫자를 제곱한다. (매핑)
result = [n**2 for n in numbers if n % 2 == 0]

print(f"최종 결과 리스트: {result}") # 출력: [4, 16, 36]
```

- 오버헤드 : 어떤 처리를 하기 위해 추가로 들어가는 시간이나 메모리 등의 비용
- 매핑 : 하나의 값을 다른 값으로 일대일 대응시켜 변환하는 과정


# 2. 데이터 바인딩과 Zip 연산
## 2.1 zip 함수의 메커니즘
- 이름 리스트, 점수 리스트처럼 서로 연관된 여러 데이터를 다루게 된다
- 이때 zip 함수는 여러 시퀀스를 병렬로 연결하여 인덱스가 같은 요소끼리 묶어주는 반복자, Iterator 를 생성한다

```
# [미션 2] 이름과 점수 리스트를 딕셔너리로 매핑
names = ['철수', '영희', '민수']
scores = [90, 85, 70]

# zip(names, scores)는 ('철수', 90), ('영희', 85)... 형태의 튜플 쌍을 생성합니다.
# dict() 생성자는 이러한 튜플 쌍을 받아 {키: 값} 형태의 해시 구조로 즉시 변환합니다.
# 이 모든 과정은 단 한 번의 순회(O(N))로 종료됩니다.
student_info = dict(zip(names, scores))

print(f"매핑된 딕셔너리: {student_info}")
# 출력: {'철수': 90, '영희': 85, '민수': 70}

# 참고: 특정 학생의 점수를 찾는 속도는 O(1)로 매우 빠릅니다.
print(f"철수의 점수: {student_info['철수']}") 
```

# 3. 집합 자료구조를 활용한 최적화
## 3.1 리스트 탐색과 집합 탐색의 차이
- 가장 중요한 대목
- 리스트에서 어떤 값이 있는지 확인하는 in 연산은 처음부터 끝까지 다 하나씩 확인하므로 O(N)이 걸린다
- 하지만 집합 Set은 내부적으로 해시 테이블을 사용한다. 따라서 값을 넣을 때 그 값의 위치, 주소를 미리 계산해두는 방식이어서 데이터가 100만개가 있어도 단 한 번에 값을 찾아내는 O(1) 의 속도가 걸린다

```
# [미션 3] 중복 제거 및 효율적 교집합 찾기
listA = [1, 5, 2, 3]
listB = [2, 3, 4, 5, 9]

# 1. 리스트를 Set으로 변환하는 과정에서 중복이 제거되고 해시 테이블이 구축됩니다.
setA, setB = set(listA), set(listB)

# 2. '&' 연산자는 집합의 교집합(Intersection)을 의미합니다.
# 내부적으로 한 집합의 요소를 꺼내 다른 집합의 해시 테이블에서 광속으로 검색합니다.
common_elements = setA & setB

# 3. 결과를 정렬된 리스트로 변환하여 반환합니다.
result = sorted(list(common_elements))

print(f"공통 요소 확인: {result}") # 출력: [2, 3, 5]
```

# 4. 해시 기반 빈도수 계산
## 4.1 빈도 분석의 비효율성 제거
- 문자열에서 각 문자의 개수를 셀 때, s.count('a') 를 반복 호출하는 건 문자열 전체를 계속해서 다시 읽기 때문에 비효율적이다
- 대신 데이터를 한 번만 읽으면서 딕셔너리에 기록해두는 방식이 훨씬 효율적이다

```
from collections import Counter

# [미션 4] 문자열 내 문자 빈도수 계산
s = "banana"

# Counter 클래스는 반복 가능한 객체를 입력받아 요소별 개수를 세어 딕셔너리 형태로 반환합니다.
# 시간 복잡도: O(N) (문자열을 단 한 번만 훑음)
char_counts = Counter(s)

print(f"문자별 등장 횟수: {dict(char_counts)}")
# 출력: {'b': 1, 'a': 3, 'n': 2}

# [심화] 가장 많이 등장한 요소 추출하기
# most_common(1)은 가장 빈도가 높은 상위 1개 요소를 반환합니다.
most_frequent = char_counts.most_common(1)
print(f"가장 많이 등장한 문자: {most_frequent[0][0]}") # 'a'
```
