## 2. 과제 수행 절차 (Assignment Workflow)
실력은 무작정 코드를 짤 때보다 '분석-설계-구현'의 단계를 밟을 때 가장 빠르게 성장합니다.


### 2.1 이번 주 미션과 테마 확인하기
매주 제공되는 [통합 가이드]를 먼저 읽어주세요. 정답을 맞히는 것도 중요하지만, 이번 주에 정복해야 할 '기술적 테마(예: 시간 복잡도 최적화, 스택/큐의 활용)'가 무엇인지 파악하는 것이 핵심입니다.


### 2.2 레퍼런스 코드를 내 것으로 만들기
Fork 해온 programmers_python_algorithm 폴더의 코드를 꼼꼼히 분석해 보세요. 직접 실행해 보며 데이터가 어떻게 변하는지 디버깅 모드로 하나하나 확인하는 과정이 원리 이해에 큰 도움이 됩니다.


### 2.3 나만의 솔루션 구현하기
이해한 내용을 바탕으로 본인의 개인 저장소(my-algorithm-solutions)에 코드를 작성합니다. 나중에 다시 봐도 가독성이 좋도록 깔끔하게 작성하고, 필요한 부분에는 주석을 달아주는 연습을 해보세요.

## 4. Git 및 GitHub 버전 관리 프로세스
코드를 올리는 과정은 단순히 파일을 전송하는 것이 아니라, 여러분의 노력을 기록으로 남기는 소중한 시간입니다. 터미널에서 아래 명령어를 차례로 입력해 볼까요?

- 작업 폴더로 이동: cd my-algorithm-solutions
- 변경 사항 등록: git add .
- 나만의 기록 남기: git commit -m "Week1: 데이터 처리 최적화 미션 완료!"
- GitHub 서버로 전송: git push origin main
